Project PHP

This is the follow-up project of Requirement Analysis (RA). Now it was time to use our practical skills and turn the theory into a web application for a client. We weren't allowed to use our own RA project, but we had used the documentary from another group. This way we learn both ways, the setup and the practical. This project was the first where I didn't make something for myself.

What I learned during this project is: communicating to our colleagues and to a client. If you can't make yourself clear, others won't be able to help you. Also, if your client doesn't understand you and vice versa the web application will not be at its full potential. Another thing I learned is to ask for help. We are learning, and we don't know everything about PHP, Laravel or AJAX. During this project I created: the login, the student and docent pages, and I took most of the styling on me.
